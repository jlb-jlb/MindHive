# Capture

Put unprocessed material in [Inbox](/0-Inbox/) when its destination is unclear.

# Review

Review [Inbox](/0-Inbox/) at least once per week. Unprocessed inbox notes older than 7 days must be routed, converted into a decision or handoff, or archived.

During review, ask:

* Is this actionable now?
* Is it a recurring responsibility?
* Is it reusable knowledge?
* Is it inactive but worth keeping?
* Does it need a decision note or handoff?

# Route

| Answer | Destination |
|---|---|
| Actionable with a finish line | [Projects](/1-Projects/) |
| Ongoing responsibility | [Areas](/2-Areas/) |
| Reusable information | [Resources](/3-Resources/) |
| Inactive context | [Archive](/4-Archive/) |
| Still unclear | [Inbox](/0-Inbox/) |

# Project Lifecycle

Projects start as `active` when they have an outcome, next action, and done condition.

Move a project to `paused` when it remains valid but should not compete for current attention.

Move a project to `done` when the done conditions are met and the result is linked from the relevant MOCs, areas, or resources.

Move a `done` project to [Archive](/4-Archive/) when it is no longer needed as active context, then set `status: archived`.

# Steer Agents

When asking an agent to work in the vault, provide:

* Desired outcome.
* Relevant links if known.
* Constraints or preferences.
* Whether to update MOCs, logs, or templates.
