# Summary

Zotero gives each account a single personal library ("My Library"); you cannot create additional personal libraries inside it. To keep an old project's papers from cluttering a new, unrelated one, use collections, group libraries, or separate profiles depending on how much isolation you need. Use this note when starting a new research topic and wanting the previous project's citations out of the way but not deleted.

# Key Points

* **Collections (recommended for most cases).** Folders inside My Library. An item can live in several collections at once without duplicating, and removing it from a collection does not delete it from the library. Archive the old project into a collection and work inside a fresh collection for the new one. Caveat: the top-level "My Library" view still pools every item together; collections are subsets, not separate spaces.
* **Group Libraries (true separation).** Genuinely separate libraries shown below My Library in the left pane. Even a private group with only you gives a fully isolated space where nothing from the old project appears. Free to create at zotero.org/groups. Trade-offs: items live in Zotero's cloud (counts against storage), and an item cannot span a group and the personal library without copying.
* **Separate profiles (maximum isolation).** Running Zotero with a second profile / data directory gives a completely independent "My Library." Cleanest separation but heaviest to manage; you switch by launching different profiles and they share nothing. Usually overkill.
* **Practical pick.** For moving to an unrelated project while keeping old citations, archive the old work into a collection and start a new collection. If old papers should vanish from view entirely, use a private Group Library.
