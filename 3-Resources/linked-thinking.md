# Principle

The value of a second brain comes from relationships. A note should reveal what it depends on, what it supports, and where a reader should go next.

# Link Types

| Link Type | Meaning | Example |
|---|---|---|
| Parent | Broader map or area | A project links to [Projects MOC](/5-MOCs/projects-moc.md). |
| Dependency | Required context or resource | A protocol links to [OKF Bundle Practice](/3-Resources/okf-bundle-practice.md). |
| Sibling | Related concept | PARA links to [Maps of Content](/3-Resources/maps-of-content.md). |
| Evidence | Source or citation | OKF practice links to [OKF Baseline](/3-Resources/okf-specification.md). |
| Continuation | Next step or handoff | A project links to a context packet. |

# Rules

* Do not rely on folder location alone to explain meaning.
* Prefer explicit links over repeated prose.
* Add links when the target changes how the current note should be interpreted.
* Keep links stable by using lowercase kebab-case file names for new notes.
